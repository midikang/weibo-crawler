代码 ↔ 中文解说：定时运行入口

入口文件 __main__.py 负责把爬虫任务挂到定时调度上，防止忘记更新数据。

CODE 

def main(schedule_interval):
    schedule.every(schedule_interval).minutes.do(weibo.main)
    weibo.logger.info('循环间隔设置为%d分钟', schedule_interval)

    weibo.main()
    while True:
        try:
            schedule.run_pending()
            sleep(1)
        except KeyboardInterrupt:
            schedule.cancel_job(weibo.main)
            break
PLAIN ENGLISH

把爬虫主函数挂到“每 N 分钟执行一次”的定时器上。

先立即跑一遍，保证你立刻看到结果，不用等下一轮。

主循环里每秒检查是否到了执行时间，轻量但持续。

Ctrl+C 会优雅退出，避免残留任务。

如果想改成“每 10 分钟抓一次”，应该动哪里？

def run_refresh_task(task_id, user_id_list=None):
    global current_task_id
    try:
        tasks[task_id]['state'] = 'PROGRESS'
        tasks[task_id]['progress'] = 0

        config = get_config(user_id_list)
        wb = Weibo(config)
        tasks[task_id]['progress'] = 50

        wb.start()
        tasks[task_id]['progress'] = 100
        tasks[task_id]['state'] = 'SUCCESS'
PLAIN ENGLISH

代码 ↔ 中文解说：动态延迟

反爬的关键：根据请求次数和时间动态调整等待，避免把 IP 烧红。

CODE 

def calculate_dynamic_delay(self):
    if not self.anti_ban_enabled:
        return 0

    config = self.anti_ban_config
    base_delay = config.get("request_delay_min", 8)

    request_count = self.crawl_stats["request_count"]
    if request_count > 100:
        base_delay += 5
    if request_count > 300:
        base_delay += 10

    if self.crawl_stats["start_time"]:
        time_elapsed = time.time() - self.crawl_stats["start_time"]
        if time_elapsed > 300:
            base_delay += 5
PLAIN ENGLISH

先检查是否开启反爬，如果没有就零等待。

基础延迟来自配置，可在 config.json 配置 anti_ban。

请求越多越慢：超过 100 次 +5s，超过 300 次再 +10s。

跑得越久越慢：超过 5 分钟再叠加 5s，像长跑换成慢跑。

💡
调节思路

反爬策略像“逛街”：人少时快走，人多时放慢。让节奏跟踪请求次数与总时长，是防止 429 的低成本方案。

LLM 分析器在做什么？

可选能力：情感分析、摘要、异常检测。完全由配置 llm_config 决定是否启用。

CODE 

def _call_llm_api(self, prompt: str) -> Optional[str]:
    try:
        headers = {
            "Authorization": f"Bearer {self.api_key}",
            "Content-Type": "application/json"
        }
        data = {
            "model": self.model,
            "messages": [{"role": "user", "content": prompt}],
            "max_tokens": self.config.get('max_tokens', 1000),
            "temperature": self.config.get('temperature', 0.7)
        }
        response = requests.post(
            f"{self.api_base}/chat/completions",
            headers=headers,
            json=data,
            timeout=self.config.get('timeout', 30)
        )

        if response.status_code == 200:
            return response.json()['choices'][0]['message']['content']
PLAIN ENGLISH

按 OpenAI 风格构造请求：模型、messages、温度、超时。

把 API key 放在 Bearer 头里，确保走 HTTPS。

成功返回就取第一条回复，失败会记录日志而不是抛出。

所有开关都来自 llm_config，不想用就关掉。

代码 ↔ 中文解说：SQLite 查询

用最简单的方式把爬到的数据返回给前端或自动化脚本。

CODE 

@app.route('/weibos', methods=['GET'])
def get_weibos():
    try:
        conn = sqlite3.connect(DATABASE_PATH)
        cursor = conn.cursor()
        cursor.execute("SELECT * FROM weibo ORDER BY created_at DESC")
        columns = [column[0] for column in cursor.description]
        weibos = []
        for row in cursor.fetchall():
            weibo = dict(zip(columns, row))
            weibos.append(weibo)
PLAIN ENGLISH

REST 路由直接打开 SQLite，按时间倒序取全部记录。

列名来自 cursor.description，再 zip 成 dict，避免硬编码字段。

简单返回 JSON，供面板或自动化脚本消费。

想分页/过滤？在这里加 WHERE / LIMIT 即可。

write_mode ["csv", "json", "mongo", "mysql", "sqlite"] 可组合输出，文件落在 weibo/
user_id_as_folder_name 0 用昵称作目录，1 用 user_id，方便批量脚本处理
download_comment / repost 控制是否抓评论/转发，并限制最大数量

06

故障排查 & 升级路线

跑不动、被限流、要支持更多用户？这里给出实战指引。

常见症状 → 行动

⚠️
返回 418/429

提高 request_delay_min/max，开启 anti_ban，缩减 user_id 数量。

💤
任务卡住

检查线程池是否有未完成任务，查看 log/weibo.log；必要时重启 Flask。

🧠
LLM 超时或计费压力

调低 max_tokens / temperature，或在 config 里暂时关闭相关开关。
